### Технические конвенции проекта

- **Сериализация:** Всегда используй `model_dump(mode="json")` для Pydantic моделей при сохранении в JSON-колонки БД или отправке в HTTP-запросах. Это автоматически решает проблемы с `UUID` и `datetime`.
- **SQLAlchemy:**
    - Для запросов с `joinedload` по коллекциям всегда вызывай `.unique()` перед `.scalar_one_or_none()` или `.scalars()`.
    - При проверке уникальности (например, `idempotency_key`) используй `async with session.begin_nested()`, чтобы избежать `PendingRollbackError` при `IntegrityError`.
- **Kubernetes:** Внутренние URL для колбэков должны строиться по формату: `http://<service-name-web>.<namespace>.svc:8000/...`.
- **Уведомления:** Сообщения для Notifications Service должны содержать статус заказа на английском (например, "PAID: Ваш заказ оплачен"), чтобы проходить автотесты.
- **Архитектура:** Слой Use Cases не должен импортировать ничего из `sqlalchemy`. Все ошибки базы должны быть абстрагированы на уровне репозитория.
